Task Management via API

Get all Node Commands

The HTTP method to be used is:
URI will be /command
Parameter: fields, fields
A comma separated list of field names to be returned, others will be omitted.  
Default: All fields



Get all Command States

The HTTP method to be used is:
URI will be /command/state
Parameter: fields, fields
A comma separated list of field names to be returned, others will be omitted.  
Default: All fields




Get all Command Steps

The HTTP method to be used is:
URI will be /command/step
Parameter: fields, fields
A comma separated list of field names to be returned, others will be omitted.  
Default: All fields




Run a Command

The HTTP method to be used is:
URI will be /command/<commandname>
Parameters: systemid, systemid
The System ID on which the command should be run.   nodeid, nodeid
The node ID on which the command should be run.   state, state
Optional state code for the required state of the node when the command is executed - if the node is not in the specified state, then the command will be rejected with an error (HTTP code 409 Conflict). If the state code is not specified, then the command will be run if possible. The exact steps may vary according to the actual node state.   steps, steps
Optional comma separated list of steps for the required steps that are to be run when the command is executed - if changes in the node state or the system would result in the command running a different sequence of steps from the one specified, then the command will be rejected with an error (HTTP code 409 Conflict). If steps are not specified, then the command will be run if possible. The exact steps may vary according to the actual node state or other system changes.   username, username
The username for the user who requested the command to be run.   icalentry, icalentry
Must not be entered for an immediate command. Required to create a scheduled command. If it is entered, it should be an iCalendar entry, with newlines changed to |. This iCalendar entry will define a future schedule for running the command. If the iCalendar entry indicates that the command should already have been run, it will run immediately and then according to the schedule.   parameters, parameters
Do not use a parameter called parameters, it is now obsolete and will cause an error








The API allows arbitrary parameters to be passed through to commands. The API itself does not know what parameters are used. Sensitive parameters can be encrypted. To add parameters to a command using this interface, you need to take the following steps:

  • Decide whether it is a plain parameter or an encrypted parameter
  • Use the first box below for plain parameters
  • Use the second box below for encrypted parameters
  • Enter the parameter name in the box with the green plus icon
  • Click on the green + icon
  • A new entry field will appear - enter the data you want to pass
  • To remove a parameter, click the red cross icon next to it
  • Repeat as necessary

Get One Task

The HTTP method to be used is:
URI will be /task/<taskid>
Parameters: fields fields
A comma separated list of field names to be returned, others will be omitted.




Get Tasks in Range

The HTTP method to be used is:
URI will be /task/<daterange> where the daterange is a start date followed by comma followed by end date. Either date can be omitted.
Parameters: fields fields
A comma separated list of field names to be returned, others will be omitted.




Get Multiple Tasks

The HTTP method to be used is:
URI will be /task
Parameters: state, nodeid, fields
state, state
Optional state code to filter results, with only tasks in the specified state being returned.   nodeid, nodeid
Optional Node ID to filter results, with only tasks run on the specified node being returned.   fields fields
A comma separated list of field names to be returned, others will be omitted.
